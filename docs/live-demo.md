# Live Demo: Claude Code → SpinChain MCP → Verified Answer

## What This Demonstrates

Claude Code generates multiple reasoning chains for a math problem, passes them to SpinChain via the MCP `optimize_reasoning` tool, and receives back the optimized fragment subset. No external API calls. Zero additional cost. The chain generation is Claude Code thinking; the optimization is local SA on the user's machine.

## The Problem

> A store sells notebooks for $3 each. If you buy 5 or more, you get a 20% discount. Maria buys 8 notebooks and her friend buys 3 notebooks in a separate transaction. How much do they pay in total?

**Correct answer:** $28.20

## The 7 Reasoning Chains

Claude Code solved the problem 7 times with different approaches:

| Chain | Approach | Answer | Correct? |
|-------|----------|--------|----------|
| 1 | Direct calculation | 28.2 | Yes |
| 2 | Discount price per unit | 28.2 | Yes |
| 3 | Step by step with verification | 28.2 | Yes |
| 4 | Working backwards from combined | 28.2 | Yes |
| **5** | **Percentage approach** | **27.2** | **No — arithmetic error: 24 × 0.8 = 18.2 (should be 19.2)** |
| 6 | Explicit percentage subtraction | 28.2 | Yes |
| 7 | Unit price focus | 28.2 | Yes |

Chain 5 contains a verifiable arithmetic error: `24 * 0.8 = 18.2` when the correct result is 19.2. This error propagates to a wrong final answer (27.2 instead of 28.2).

## The MCP Call

```
optimize_reasoning(
  completions=[chain1, chain2, ..., chain7],
  num_reads=100,
  num_sweeps=1000,
  similarity_threshold=0.9
)
```

Called directly from Claude Code via the registered SpinChain MCP server. No API key, no network call, no cost.

## SpinChain's Output

**Input:** 7 chains → 33 unique fragments after deduplication

**Selected:** 17 fragments from the correct chains

**Excluded:** Fragments from Chain 5 (the erroneous chain), including `24 * 0.8 = 18.2`

**Solver:** All 100 SA reads converged to the same energy (-103.38) — zero ambiguity in the selection.

### Selected fragments (all verified-correct arithmetic):

```
Maria buys 8 notebooks at $3 each.
8 * 3 = 24.                           ← verified: 8×3 = 24 ✓
She gets 20% discount because 8 >= 5.
Discount: 24 * 0.2 = 4.8.             ← verified: 24×0.2 = 4.8 ✓
Maria pays 24 - 4.8 = 19.2.           ← verified: 24−4.8 = 19.2 ✓
Friend buys 3 at $3 each.
3 * 3 = 9.                            ← verified: 3×3 = 9 ✓
No discount (3 < 5).
Total: 19.2 + 9 = 28.2.              ← verified: 19.2+9 = 28.2 ✓
The answer is 28.2.
```

### Excluded fragment (from Chain 5):

```
24 * 0.8 = 18.2                       ← WRONG: 24×0.8 = 19.2, not 18.2
```

## What This Proves

1. **The full loop works end-to-end.** Claude Code generates chains → SpinChain MCP tool processes them → optimized fragments returned. No external infrastructure needed.

2. **Fragment-level selection beats chain-level selection.** SpinChain didn't just pick "the best chain" — it selected individual fragments across multiple chains, excluding the erroneous fragment from Chain 5 while keeping the correct structural reasoning from other chains.

3. **Arithmetic verification detects the error.** The fragment `24 * 0.8 = 18.2` is verifiable as incorrect by parsing the expression and evaluating it. The QUBO's verification terms (φ, ψ) raise the energy of fragments from chains containing arithmetic errors.

4. **Zero marginal cost.** The reasoning chains are generated by Claude Code as part of its normal operation. The SpinChain optimization runs locally. No additional API calls, no token cost, no latency beyond the SA solve (~50ms).

## Comparison: SpinChain vs. Majority Vote

On this problem, majority vote also produces 28.2 (6:1 majority). Both methods get the right answer. The difference is in the mechanism:

| Aspect | Majority Vote | SpinChain |
|--------|--------------|-----------|
| How it decides | Counts final answers (6 say 28.2, 1 says 27.2) | Evaluates individual reasoning fragments |
| What it outputs | One answer (28.2) | 17 verified fragments with full reasoning trace |
| Error detection | None — doesn't know Chain 5 has an error | Detects `24 * 0.8 = 18.2` as arithmetic error |
| When it fails | 4:3 split where majority is wrong | Only when arithmetic errors are undetectable (interpretation errors) |

The value gap widens on harder problems where the answer split is closer (e.g., 4:3) and the majority's error is arithmetically detectable. In those cases, majority vote follows the wrong majority while SpinChain detects the error and selects the correct minority.

## How to Reproduce

Register SpinChain as an MCP server in Claude Code, then ask Claude to solve a math problem multiple times with different approaches and pass the results to `optimize_reasoning`. The demo requires no API key beyond what Claude Code already uses.

```json
{
  "mcpServers": {
    "spinchain": {
      "type": "stdio",
      "command": "uv",
      "args": ["run", "--directory", "/path/to/spinchain", "python", "-m", "spinchain"]
    }
  }
}
```
